# Documentation Guide

This repository now has two documentation layers:

- `docs/`: maintainer and contributor documentation
- `website/src/content/docs/`: user-facing product documentation rendered by the website

## Maintainer docs

Use `docs/` for:

- build and test workflows
- release and packaging notes
- architecture details
- internal feature planning

## User docs

Use `website/src/content/docs/` for:

- onboarding guides
- feature walkthroughs
- workflow explanations
- discoverability of hidden product behavior

These files are written in Markdown and rendered on the public site.
